# Iterator 迭代器

道理是这样的： 实现了 @@iterator 接口的对象就是一个迭代器对象 , 此对象可以被 for of 迭代，可以由 Array.from 生成数组

比如我们用 for of 来迭代数组，但是 for of 却不能迭代普通的对象

// 迭代数组
const arr: Array<Number> = [1, 2, 3]

for (const item of arr) {
	console.log(item) // 1, 2, 3
}

//迭代对象
const obj = {
	a: 1,
	b: 2,
}

for (const item of obj) {
	// Error -> 类型“{ a: number; b: number; }”必须具有返回迭代器的 "[Symbol.iterator]()" 方法。
	console.log(item)
}

这就是由于 Array.prototype 上实现了迭代器的 @@iterator 方法， 故可以由 for of 迭代

其他实现迭代器接口 @@iterator 的内置类型的还有

  • Array.prototype[@@iterator]()
  • TypedArray.prototype[@@iterator]()
  • String.prototype[@@iterator]()
  • Map.prototype[@@iterator]()
  • Set.prototype[@@iterator]()

所以这些类型的实例出来的对象 都可以被 for of , Array.from 迭代

// 比如字符串
const msg: string = 'hello'

for (let item of msg) {
	console.log(item) // h, e, l, l, o
}

Array.from(msg) // [ "h", "e", "l", "l", "o" ]

也就是说如果想让一个对象可迭代就必须实现 @@iterator 方法, 那么来try一哈

class List implements Iterable<Number> {
  private data: Array<Number>

  constructor (data) {
    this.data = data
  }

  [Symbol.iterator] () {
    let index = 0
    const len = this.data.length
    
    return {
      next: () => {
        const value = this.data[index]

        if (index <= len - 1) {
          index ++
          return { value, done: false }
        } else {
          return { value: 0, done: true }
        }
      }
    }
 }
}

const list = new List([1, 2, 3])

for (const item of list) {
	console.log(item) // 1, 2, 3 、 list对象成功被 for of 迭代
}
